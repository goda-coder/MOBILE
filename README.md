# Wallet Mobile App (Flutter)

A native iOS + Android mobile app for the Wallet platform — same backend, same features as the web frontend, optimised for phones. Matches the magenta/pink dark-fintech design language.

## Stack

| Concern        | Choice                                        |
|----------------|-----------------------------------------------|
| Framework      | Flutter 3.22+ / Dart 3.4+                     |
| State          | Riverpod 2 (`flutter_riverpod`)               |
| Routing        | go_router 14                                  |
| HTTP           | dio 5                                         |
| Secure storage | flutter_secure_storage (Keychain / Keystore)  |
| Camera         | `camera` package (KYC + liveness)             |
| QR generation  | `qr_flutter`                                  |
| QR scanning    | `mobile_scanner` (MLKit / Vision)             |
| Typography     | Google Fonts — Jost + Overpass Mono           |

No platform channels are written by hand. Everything goes through well-maintained pub packages.

## ⚠️ Bootstrap step

This package ships **only the `lib/` source and `pubspec.yaml`** — the platform-specific folders (`android/`, `ios/`, `web/`, `windows/`, `macos/`, `linux/`) **are not included** because they are machine-generated by the Flutter toolchain and contain user-specific signing config, bundle IDs, etc.

Before running, scaffold them from this directory:

```bash
cd mobile
flutter create . \
  --org com.example.wallet \
  --project-name wallet \
  --platforms=android,ios
# Add ",web,macos,windows,linux" if you want desktop / PWA targets too.
```

`flutter create` is non-destructive — it only writes missing files. Your `lib/`, `pubspec.yaml`, and `analysis_options.yaml` stay exactly as shipped.

Then:

```bash
flutter pub get
flutter run
# or
flutter run --dart-define=API_BASE_URL=https://api.example.com
```

The default `API_BASE_URL` is `http://10.0.2.2:8081` — the Android emulator's special address for "host machine on port 8081". The biometric payment and WebSocket URLs are derived from this value automatically, so a single `--dart-define` covers everything.

| Target           | What to pass                                         |
|------------------|------------------------------------------------------|
| Android emulator | default works                                        |
| iOS simulator    | `--dart-define=API_BASE_URL=http://localhost:8081`   |
| Physical device  | `--dart-define=API_BASE_URL=http://<your-LAN-ip>:8081` |
| Production       | `--dart-define=API_BASE_URL=https://api.example.com` |

## Platform permissions (one-time setup)

After `flutter create`, add the following to the generated platform files. Skip them and the camera / QR / KYC features will silently fail at runtime.

### Android — `android/app/src/main/AndroidManifest.xml`

Add inside `<manifest>`, **before** `<application>`:

```xml
<uses-permission android:name="android.permission.CAMERA" />
<uses-permission android:name="android.permission.INTERNET" />
<uses-feature android:name="android.hardware.camera" android:required="false" />
<uses-feature android:name="android.hardware.camera.autofocus" android:required="false" />
```

`mobile_scanner` requires `minSdkVersion >= 21`. Open `android/app/build.gradle` (or `build.gradle.kts`) and bump if needed:

```gradle
defaultConfig {
    minSdkVersion 21
    targetSdkVersion 34
}
```

### iOS — `ios/Runner/Info.plist`

Add inside the top-level `<dict>`:

```xml
<key>NSCameraUsageDescription</key>
<string>The wallet uses your camera for identity verification and to scan QR codes for payments.</string>
<key>NSMicrophoneUsageDescription</key>
<string>Microphone access isn't actively used but is requested by the camera package.</string>
<key>io.flutter.embedded_views_preview</key>
<true/>
```

`mobile_scanner` requires iOS deployment target **11.0+**. Open `ios/Podfile` and set:

```ruby
platform :ios, '11.0'
```

Then from `ios/`: `pod install`.

## Auth model

1. Login hits `POST /api/v1/auth/login`, which returns `{ accessToken, refreshToken, role, email, userId }`.
2. Tokens are written to the device's **secure storage** (iOS Keychain / Android Keystore via `flutter_secure_storage`'s `encryptedSharedPreferences`).
3. Every request passes through a Dio `_AuthInterceptor` that:
   - Injects the bearer token.
   - On `401`, performs one refresh-token rotation.
   - On refresh failure, clears state — go_router's `refreshListenable` immediately redirects to `/login`.

## Camera & liveness

**CameraCapture** (`widgets/camera_capture.dart`) — full-screen preview with an optional pulsing face-guide oval. Mirrors the front camera so it feels like a real mirror. Returns a `File` containing a JPEG still.

**LivenessCapture** (`widgets/liveness_capture.dart`) — for active liveness checks. Displays an instruction (e.g. "Blink slowly, twice"), captures ~24 stills over 4 seconds at 6 fps via `takePicture()`, base64-encodes them, and posts to `/api/v1/kyc/liveness/verify`. Temporary files are deleted as they are consumed.

**AppQrScanner** (`widgets/qr_scanner.dart`) — wraps `mobile_scanner` with a brand-coloured corner-bracket viewfinder. Fires exactly once on first decode, then stops.

The KYC flow handles permission denial, restricted access, and missing cameras gracefully, showing human-readable error messages instead of raw platform errors.

## Design

- **Palette** — `#C800DF → #E60076 → #FF4D8D` gradient stops over an `ink-950` (`#08070C`) surface ladder.
- **Typography** — Jost everywhere via `google_fonts`. Overpass Mono with `FontFeature.tabularFigures()` (via the `numTextStyle()` helper) for money columns.
- **Layout** — bento-style cards, soft brand glow behind the balance, role-aware bottom nav.
- **Motion** — `AnimationController` pulse on the face-guide oval, gradient progress bar on the KYC wizard, `LinearProgressIndicator` for the 4-second liveness capture.

## What's not included

| Feature | Notes |
|---|---|
| Real-time notifications | Needs a SignalR hub on the backend first. The `/notifications` page is wired but disconnected. |
| Push notifications | Add `firebase_messaging` + your Firebase project when ready. |
| Biometric unlock | `local_auth` would slot in cleanly — omitted to keep the dependency tree small. |
| Internationalization | English-only. Add `flutter_localizations` + `intl` ARBs for Arabic. |
| Tests | None included. Add `widget_test.dart` and golden tests under `test/` once the design is stable. |
| Deep links from Paymob | Requires universal-links / app-links plumbing on each platform. |

## Web vs mobile — which to keep?

You can run both simultaneously:

- **Mobile (this folder)** — the customer-facing app. Phone-first, native camera, Material 3, Riverpod.
- **Web (`../frontend/`)** — useful as an **admin desktop view**. Large screens work better for triaging KYC queues, managing payments, and report dashboards.

Route customers to the mobile app stores and admins to the web URL. If you want mobile-only, deleting `../frontend/` is safe — nothing in the backend depends on it.

## Troubleshooting

| Symptom | Fix |
|---|---|
| `Connection refused` on Android emulator | Backend isn't running on `10.0.2.2:8081`. Check `docker compose ps`. |
| `Connection refused` on iOS simulator | Pass `--dart-define=API_BASE_URL=http://localhost:8081`. |
| Camera shows black on iOS simulator | Camera doesn't work in iOS simulators. Use a physical iPhone. |
| `Camera permission was denied` | Settings → your app → Camera → toggle on. |
| `mobile_scanner` build fails on Android | Bump `minSdkVersion` to 21 in `android/app/build.gradle`. |
| `mobile_scanner` build fails on iOS | Set platform to `11.0` in `ios/Podfile`, then `cd ios && pod install`. |
| Fonts look wrong | `google_fonts` downloads on first run and requires internet on first launch. To bundle fonts offline, see the package README. |
